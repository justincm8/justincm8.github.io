Static Data Structures

  • The arrays that we have seen so far are static data structures.
  • That means their size (the number of elements they can contain) can not be changed after they are created.
  • By contrast, a dynamic data structure can change size.

    • Lists

  • You can think of a list as a dynamic array; its size can change.

  • We could actually omit the type parameter on the right side of the assignment. Java can infer it from the left side.

    • List vs. ArrayList

  • Okay, why is there List on the left side of the assignment but ArrayList on right side?
  • List refers to an interface, a guide that determines the operations we can perform on a List, like adding and removing elements, but does not contain any actual implementation.
  • That is, the List interface defines what a List can do (what methods it should have) but not how it will do it.
  • On the other hand, ArrayList defines an actual implementation of a List (using an array).
  • That is, under the hood, a List uses an array to store elements.
  • Using an array isn't the only way to implement a list as we'll see later.

    • Adding Elements to a List

  • We can use add to add elements to the end of a List, increasing its size.

    List Get and Set

  • We can't use bracket notation (like names[i]) with Lists.
  • Instead we use get and set methods.
  • Note that the set method takes two arguments: the index to set and the value to set it to.

    • remove

  • The are two ways we can remove an element from a list:
  • First, specifying an index:

    • remove

  • Second, by specifying the element we want to remove.
  • If you try to remove an element that is not in the List, nothing will happen.
  • remove actually returns a boolean representing whether the removal was successful or not.

    • Boxing

  • What if we want our list to store integers? Notice that doing List<int> fails:
  • This is because a List can only store objects but int is a primitive type.

    • Boxing

  • So, in order to store integers, we have to use the Integer class, which "boxes" the int primitive type into an object.
  • There are "boxing" classes for all the primitive types, including Double, Character, Boolean, and Float.
