Linked Data Structures

  • In our dynamic, linked data structures that we have seen so far (linked list), each node had a reference to just one other node.

    • Trees

  • We can generalize this idea by allowing nodes to be connected to multiple other nodes.
  • If these connections do not form a loop, the data structure is called a tree.
  • This is a special case of the more general data structure graphs, which do allow for loops.

    • What can Trees represent?

  • A file system on a computer.
  • The Java Class Hierarchy

    • Binary Trees

  • A binary tree is a special case of a tree where each node has at most two children: a left-child and a right-child.
  • If node A has node B has a child, node A is said to be node B's parent.
  • The highest node in the hierarchy is the root node of the tree.
  • Nodes that have no children are said to be leaf nodes.
  • A subtree is a smaller tree that is part of a larger tree.
